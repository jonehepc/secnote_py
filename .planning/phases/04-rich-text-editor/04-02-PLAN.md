<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 用户点击格式工具栏 → QTextDocument | 用户操作修改富文本文档格式，必须通过 Qt 文档模型而非手写 HTML。 |
| QColorDialog → QTextCharFormat | 系统颜色对话框返回外部 UI 输入，必须只转换为 QColor 格式属性。 |
| QTextEdit HTML → SNoteItem.content | 本计划只发出 HTML 字符串，不写明文临时文件。 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Tampering | RichTextEditorWidget character actions | mitigate | 所有字符格式通过 `QTextCharFormat` + `mergeCurrentCharFormat` 应用，避免手写 HTML 拼接破坏文档结构。 |
| T-04-06 | Information Disclosure | RichTextEditorWidget.load_html | mitigate | `load_html` 阻断 signals 并清理 undo/redo stacks，避免加载内容误置脏或跨页面残留。 |
| T-04-07 | Spoofing | Color buttons | accept | 系统 `QColorDialog` 是本地桌面可信 UI；颜色只影响文档格式，不执行外部代码。 |
| T-04-08 | Denial of Service | Font list | accept | `QFontComboBox` 枚举系统字体由 Qt 管理；不加载外部字体文件。 |
</threat_model>
