## Tests

### 1. 标签栏展示与位置
expected: 打开一个含页面的笔记本并选中页面后，右侧编辑区顶部应显示标签栏，位置在格式工具栏和正文编辑区上方。已有标签显示为 chip；没有标签时显示空态/添加入口，不应挤压或隐藏正文编辑区。
result: pass

### 2. 添加标签校验与 dirty 状态
expected: 在标签输入框输入合法标签并点击“添加标签”或回车后，新标签应立即以 chip 形式出现，状态栏给出中文反馈，当前笔记本变为未保存状态；空白、超过 32 字符或与当前页面大小写不敏感重复的标签应被拒绝并显示校验文案。
result: pass

### 3. 移除标签与补全候选
expected: 点击某个标签 chip 的移除入口后，该标签应从当前页面消失并标记未保存；新增/已有标签应作为补全候选出现，补全大小写不敏感，切换笔记本后不应显示旧笔记本的标签候选。
result: pass

### 4. 标签保存与重新打开
expected: 添加或移除标签后保存笔记本，关闭并重新打开同一加密笔记本时，页面标签应保持保存后的状态；标签不应出现在正文富文本内容中，也不应改变页面列表标题显示。
result: pass

### 5. 搜索弹窗入口与生命周期
expected: 打开笔记本后，编辑菜单中应有“搜索(&F)...”入口，Ctrl+F 也应打开同一个非模态搜索弹窗；未打开笔记本时搜索入口应不可用。搜索弹窗打开后主窗口仍可操作，重复触发应复用/聚焦同一个弹窗。
result: pass

### 6. 搜索字段与结果展示
expected: 在搜索弹窗输入关键词并点击搜索或回车后，应按勾选字段搜索标题、正文和可选标签，显示结果数量、标题、分区路径和高亮片段；空查询应显示提示且不搜索，无结果应显示无结果文案。
result: pass

### 7. 搜索安全片段
expected: 搜索正文内容时，结果片段应显示从富文本正文提取出的纯文本并只用受控高亮标记突出关键词；原始 HTML、script/style 或事件属性不应在结果中作为可执行或原始标记出现。包含 C++、[abc]、a.b、foo(bar) 这类特殊字符的关键词应按字面量匹配。
result: pass

### 8. 搜索结果跳转
expected: 点击或激活搜索结果后，主窗口应展开并选中对应分区与页面，右侧编辑器加载该页面内容；搜索弹窗应保持打开，跳转不应把笔记本标记为未保存。若结果对应页面已不存在，应显示固定中文提示而不崩溃。
result: pass

### 9. 标签字段搜索协同
expected: 仅勾选标签字段并搜索某个标签文本时，应命中带有该标签的页面；激活该搜索结果应跳转到对应页面，同时弹窗保持打开并且笔记本 dirty 状态不变化。
result: pass
