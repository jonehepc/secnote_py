<feature>
  <name>SectionFilterProxy 分区树过滤</name>
  <files>
    - src/secnotepad/model/section_filter_proxy.py (NEW)
    - tests/model/test_section_filter_proxy.py (NEW)

  <behavior>
    核心行为 — filterAcceptsRow 仅接受 section 类型节点：
    - 场景 1 (仅 section): source_index.internalPointer().item_type == "section" → filterAcceptsRow 返回 True
    - 场景 2 (note 节点): source_index.internalPointer().item_type == "note" → filterAcceptsRow 返回 False
    - 场景 3 (递归过滤): 嵌套的 section 子节点（例如: 工作 → 项目A → 子分区）由 setRecursiveFilteringEnabled(True) 递归评估，所有 section 可见
    - 场景 4 (note 子节点不自动接受): 即使父节点是 section，子 note 节点不会因 autoAcceptChildRows=True 而泄露——因为显式设置 setAutoAcceptChildRows(False)
    - 场景 5 (源模型变更传播): 向 TreeModel 添加新 section 后，代理模型自动反映变更，rowCount 增加
    - 场景 6 (mapToSource 正确): proxy.index(row, 0).mapToSource() 返回源模型中对应的 QModelIndex
    - 场景 7 (sourceModel 为 None 防御): filterAcceptsRow 在 sourceModel() is None 时返回 False（不崩溃）

    边界用例：
    - 空树（只有根分区无子节点）：代理 rowCount 为 0
    - 只有 note 无 section：代理 rowCount 为 0
    - 深层嵌套（3+ 层 section）：所有 section 可见，中间 note 不可见
  </behavior>
  <implementation>
    实现 SectionFilterProxy 类：

    1. 继承 QSortFilterProxyModel
    2. __init__ 中：
       - 调用 super().__init__(parent)
       - self.setRecursiveFilteringEnabled(True)   # Qt6: 递归评估所有子节点
       - self.setAutoAcceptChildRows(False)         # 关键: 防止 note 子节点泄露

    3. filterAcceptsRow(source_row, source_parent):
       - source = self.sourceModel()
       - 如果 source is None → 返回 False（防御性检查，对应 Open Question #1）
       - 调用 source.index(source_row, 0, source_parent) 获取源索引
       - 如果索引无效 → 返回 False
       - item = index.internalPointer()
       - 返回 item.item_type == "section"

    文件组织遵循 tree_model.py 的代码风格：
    - """模块 docstring 中文，引用决策编号 D-49"""
    - 导入块：from PySide6.QtCore import QSortFilterProxyModel, QModelIndex
    - 从 .snote_item import SNoteItem（仅用于类型提示）

    注意 (Pitfall 1): filterAcceptsRow 使用源模型索引和源行号——不需要 mapToSource/mapFromSource。
    这些映射方法由 QSortFilterProxyModel 基类自动提供，用于 Proxy ↔ Source 索引转换，
    在 Phase 03 Plan 03/04 的 MainWindow 中使用。
  </implementation>
