# 需求定义：SecNotepad

**定义日期：** 2026-05-07
**核心价值：** 笔记内容始终以加密状态保存在磁盘上，只有持有密钥的用户才能解密和阅读。

## v1 需求

当前版本要实现的需求。每条需求将映射到 Roadmap 的阶段中。

### 文件操作

- [ ] **FILE-01**：用户可以新建空白笔记本，在未保存前内容仅存在于内存中
- [ ] **FILE-02**：用户可以打开本地 `.secnote` 文件，选择文件后输入密钥进行解密加载
- [ ] **FILE-03**：用户可以保存笔记本，若已有路径则直接加密写入，若无路径则弹出保存对话框
- [ ] **FILE-04**：用户可以将笔记本另存为新文件，指定新路径和新密钥
- [ ] **FILE-05**：用户关闭笔记本或退出应用时，若有未保存更改则提示保存

### 加密

- [ ] **CRYPT-01**：新建或另存时，用户需输入加密密钥（密码），支持显示/隐藏切换
- [ ] **CRYPT-02**：打开笔记本时，用户需输入密钥解密，密钥错误时显示错误提示且无法加载
- [ ] **CRYPT-03**：保存时使用 SHA-512 派生密钥 + AES-256-CFB 加密整体 JSON 数据写入文件
- [ ] **CRYPT-04**：提供密码生成工具，可按指定长度生成随机密码

### 导航

- [ ] **NAV-01**：分区以树形结构展示，支持多级嵌套（父分区 → 子分区）
- [ ] **NAV-02**：页面在所属分区下以列表形式展示，点击分区切换页面列表
- [ ] **NAV-03**：用户可以创建、重命名、删除分区（删除分区时级联删除子分区和页面）
- [ ] **NAV-04**：用户可以在当前选中的分区下创建、重命名、删除页面

### 富文本编辑

- [ ] **EDIT-01**：页面内容使用 QTextEdit 进行富文本编辑，内容以 HTML 格式存储
- [ ] **EDIT-02**：支持字体格式：加粗、斜体、下划线、删除线
- [ ] **EDIT-03**：支持字体选择（系统字体列表）和字号调整
- [ ] **EDIT-04**：支持文字颜色和背景颜色设置
- [ ] **EDIT-05**：支持段落对齐（左对齐、居中、右对齐、两端对齐）
- [ ] **EDIT-06**：支持列表（无序列表、有序列表）和待办事项复选框
- [ ] **EDIT-07**：支持标题样式（H1 - H6）
- [ ] **EDIT-08**：支持缩进调整（增加/减少缩进）
- [ ] **EDIT-09**：支持撤销和重做操作
- [ ] **EDIT-10**：支持剪切、复制、粘贴操作
- [ ] **EDIT-11**：支持页面缩放（放大/缩小文字显示）

### 标签

- [ ] **TAG-01**：用户可以为一个页面添加多个标签
- [ ] **TAG-02**：用户可以移除页面的标签
- [ ] **TAG-03**：标签在页面列表或页面上可见展示

### 搜索

- [ ] **SRCH-01**：用户可以通过关键词搜索页面标题和内容
- [ ] **SRCH-02**：搜索结果以列表展示，包含标题和高亮匹配片段
- [ ] **SRCH-03**：点击搜索结果可跳转到对应页面

### 界面

- [ ] **UI-01**：主窗口采用分区树（左侧）+ 页面列表（中间）+ 编辑区（右侧）的三栏布局
- [ ] **UI-02**：工具栏提供常用操作的图标按钮（新建、打开、保存、另存为等）
- [ ] **UI-03**：富文本格式工具栏在编辑区上方，选中页面后可用
- [ ] **UI-04**：应用使用现代化 Qt 样式表（QSS）进行视觉美化
- [ ] **UI-05**：状态栏显示当前操作反馈信息
- [ ] **UI-06**：窗口标题栏显示应用名称和当前文件路径，未保存时带 `*` 标记

## v2 需求

推迟到未来版本，不在当前 Roadmap 中。

### 云端同步

- **SYNC-01**：用户可将笔记同步到远程服务器
- **SYNC-02**：支持从远程服务器打开笔记

### 高级编辑

- **EDIT-V2-01**：支持插入图片并嵌入笔记内容
- **EDIT-V2-02**：支持表格编辑

## 排除范围

| 功能 | 原因 |
|------|------|
| 手写/绘图 | 用户明确不需要 |
| 云同步/多设备同步 | 本版本聚焦本地使用 |
| 多用户协作 | 个人使用场景 |
| 移动端 | 仅桌面平台 |
| 第三方登录 | 仅本地密钥认证 |
| 插件系统 | 保持简单，避免过度设计 |
| Markdown 编辑模式 | 富文本编辑已满足需求 |
