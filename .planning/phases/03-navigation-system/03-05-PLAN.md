    在 `src/secnotepad/ui/main_window.py` 中恢复页面列表 Ctrl+N，但必须避免与主窗口 `_act_new` 的 Ctrl+N 冲突。

    推荐实现（优先使用）：
    1. 在顶部 QtGui 导入中加入 `QShortcut`（如果尚未导入）：`from PySide6.QtGui import QAction, QKeySequence, QCloseEvent, QShortcut`。
    2. 在 `MainWindow.__init__` 的导航状态变量区域添加：`self._shortcut_new_page: QShortcut = None`。
    3. 在 `_setup_navigation_shortcuts()` 末尾创建页面列表专属快捷键：
       ```python
       self._shortcut_new_page = QShortcut(QKeySequence("Ctrl+N"), self._list_view)
       self._shortcut_new_page.setContext(Qt.WidgetShortcut)
       self._shortcut_new_page.activated.connect(self._on_new_page)
       ```
       关键要求：parent 必须是 `self._list_view`，context 必须是 `Qt.WidgetShortcut`。这样只有 `_list_view` 获得焦点时才激活，不与 MainWindow 的 `_act_new` 形成应用级歧义。
    4. 保留主窗口 `_act_new` 的 Ctrl+N 新建笔记本行为，不要改掉菜单/工具栏的新建笔记本快捷键。
    5. 在 `_teardown_navigation()` 中清理旧快捷键，确保重复 `_setup_navigation()` 后不重复触发：
       ```python
       if self._shortcut_new_page is not None:
           self._shortcut_new_page.activated.disconnect(self._on_new_page)
           self._shortcut_new_page.setParent(None)
           self._shortcut_new_page.deleteLater()
           self._shortcut_new_page = None
       ```
       disconnect 需要包在 `try/except (RuntimeError, TypeError)` 中，匹配现有 teardown 风格。
    6. 如果选择 QAction 而非 QShortcut，则必须设置 `self._act_new_page.setShortcutContext(Qt.WidgetShortcut)` 并只 add 到 `_list_view`；同时在 `_teardown_navigation()` remove/delete。不要使用 WindowShortcut/ApplicationShortcut。
    7. 调整 Task 1 的 `test_ctrl_n_shortcut_is_scoped_to_page_list` 以精确匹配最终实现名称；推荐最终测试断言 `_shortcut_new_page.parent() is _list_view` 和 `_shortcut_new_page.context() == Qt.WidgetShortcut`。
  </action>
