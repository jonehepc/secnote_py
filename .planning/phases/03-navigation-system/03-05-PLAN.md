<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 键盘输入 → 页面 CRUD handler | 用户按键跨越 UI 事件边界触发 `_on_new_page()` 数据变更 |
| 主窗口快捷键 → 页面列表快捷键 | 同一键序列在不同 widget/context 之间分发，可能误触发错误动作 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Tampering | Ctrl+N 页面快捷键 | mitigate | 快捷键只调用现有 `_on_new_page()`；该 handler 通过 PageListModel.add_note 更新当前分区并调用 `mark_dirty()`。 |
| T-03-08 | Denial of Service | 重复 `_setup_navigation()` 后重复快捷键绑定 | mitigate | 在 `_teardown_navigation()` 中 disconnect/delete `_shortcut_new_page`，并用行为测试覆盖重复初始化后 Ctrl+N 只创建一个页面。 |
| T-03-09 | Spoofing | Ctrl+N 键序列同时代表“新建笔记本”和“新建页面” | mitigate | 页面快捷键使用 `Qt.WidgetShortcut` 且 parent 为 `_list_view`，只有页面列表聚焦时触发；主窗口 `_act_new` 保持原快捷键。 |
</threat_model>
