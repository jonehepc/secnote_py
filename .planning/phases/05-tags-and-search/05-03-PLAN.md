<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TagBarWidget → MainWindow | UI 信号 payload 进入当前页面 `SNoteItem.tags` |
| SNoteItem.tags → Serializer/FileService | 标签元数据随整体 JSON 进入现有加密保存链路 |
| 页面选择 → 标签栏刷新 | 当前页面切换时解密内存数据展示到 UI |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03-01 | T | `_on_tag_added` | mitigate | MainWindow handler 复核 trim、长度 32、casefold 去重，不能只信任 widget。 |
| T-05-03-02 | R/T | dirty-state | mitigate | 成功 add/remove 后调用 `mark_dirty()`；重复/无效输入不置脏；测试断言 `_is_dirty`。 |
| T-05-03-03 | I | persistence | mitigate | 不新增明文 tags 文件；标签只通过 `Serializer.to_json()` 进入现有加密 `.secnote` 保存。 |
| T-05-03-04 | T/I | content boundary | mitigate | 不写 `SNoteItem.content`，不把标签插入富文本 HTML；grep 验证 `note.content` 与 tag 不混用。 |
| T-05-03-05 | I | session leakage | mitigate | `_collect_available_tags()` 每次从当前 `_root_item` 收集，打开/新建后不保留旧文件候选。 |
</threat_model>
