# Phase 02 — UI Design Contract

> Visual and interaction contract for desktop Qt/PySide6 phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Qt native widgets — no CSS framework) |
| Preset | not applicable |
| Component library | Qt6 Widgets (QDialog, QLineEdit, QPushButton, QCheckBox, QProgressBar, QLabel, QListWidget, QComboBox) |
| Icon library | Qt built-in icons (QStyle.StandardPixmap) + custom SVG assets if needed |
| Font | Qt system default (follows desktop environment theme) |
| Platform | Python 3 + PySide6 (Qt6), Windows 10+ / Linux |

---

## Spacing Scale

This phase uses Qt default layout spacing exclusively. No custom spacing tokens are specified.

| Token | Qt Default | Notes |
|-------|-----------|-------|
| Layout margin | ~11px (Qt default) | QDialog / QVBoxLayout / QHBoxLayout default margins |
| Widget spacing | ~6px (Qt default) | Default spacing between child widgets in a layout |
| Form field spacing | ~6px (Qt default) | Between label and input in form layouts |
| Button spacing | ~6px (Qt default) | Between buttons in button rows (QDialogButtonBox) |

Exceptions: none. All dialogs and windows in Phase 2 use Qt default spacing.

**Phase 1 precedent:** WelcomeWidget already uses Qt default layout margins (inherited from QWidget layout). Phase 2 follows same pattern.

---

## Typography

All text elements use Qt system default font (follows OS/DE theme settings).

| Role | Qt Default | Notes |
|------|-----------|-------|
| Body / Labels | System default (~13px on most platforms) | All dialog text, labels, list items |
| Password dialog heading | System default, bold | For dialog title (window title bar) or section labels |
| Password strength label | System default | "弱" / "中" / "强" labels use same body size |
| Password generator heading | System default, bold | Sub-dialog title |

No custom font sizes, weights, or line-heights are specified. Qt handles all rendering natively.

**Phase 1 precedent (for reference):** WelcomeWidget heading uses 24pt bold, subtitle uses 12pt. This is specific to the welcome page and does not carry into Phase 2 dialog UIs.

---

## Color

This phase uses Qt system default colors exclusively. No QSS stylesheets are applied.

| Role | Source | Usage |
|------|--------|-------|
| All surfaces | Qt system palette (QPalette) | Dialog backgrounds, window backgrounds |
| All text | Qt system palette (QPalette) | Labels, input text, error messages |
| All input fields | Qt system palette (QPalette) | QLineEdit backgrounds and text |
| All buttons | Qt system palette (QPalette) | QPushButton, QCheckBox default styling |
| Error text | Qt system palette, QPalette.ColorRole.WindowText (or explicit red via palette) | Password mismatch warning, decrypt failure message |
| Password strength indicator | Application-defined: low=red, medium=orange, high=green | QProgressBar styling for strength bar |

**No custom QSS in Phase 2.** QSS styling is deferred entirely to Phase 6 (UI-04).

## Interaction Design

### Dialog: PasswordDialog (新建/首次保存)

**Mode:** `SET_PASSWORD`

| Element | Type | Behavior |
|---------|------|----------|
| Label: "密码" | QLabel | Static text above first password field |
| Password input 1 | QLineEdit + QAction (eye icon) | EchoMode=Password, toggleable via eye icon |
| Label: "确认密码" | QLabel | Static text above second password field |
| Password input 2 | QLineEdit + QAction (eye icon) | EchoMode=Password, toggleable via eye icon |
| Password mismatch warning | QLabel | Red text, shown only when contents differ and non-empty |
| Password strength bar | QProgressBar | 0-100 range, color changes: red (0-33), orange (34-66), green (67-100) |
| Strength text | QLabel | Shows "弱" / "中" / "强" above or beside bar |
| "生成密码" button | QPushButton | Opens PasswordGenerator sub-dialog |
| Confirm button | QPushButton | Disabled when passwords empty or mismatch; enabled when both match and >= 8 chars |
| Cancel button | QPushButton | Calls reject(), dialog closes |

**States:**

| State | Password 1 | Password 2 | Warning | Strength Bar | Confirm Button |
|-------|-----------|-----------|---------|-------------|---------------|
| Initial (empty) | Empty | Empty | Hidden | Hidden | Disabled |
| Filling P1 only | Typing | Empty | Hidden | Updates live | Disabled |
| P2 typing, different | Content | Content (differs) | Red: "两次输入的密码不一致" | Live | Disabled |
| P2 matches | Content | Content (matches) | Hidden | Live | Disabled if < 8 chars; Enabled if >= 8 |
| After generator | Generated pwd | Same as P1 | Hidden | Strong | Enabled |
| Error on save | Retained | Retained | Hidden | Live | Enabled (if valid) |

### Dialog: PasswordDialog (打开文件)

**Mode:** `ENTER_PASSWORD`

| Element | Type | Behavior |
|---------|------|----------|
| Label: "请输入文件加密密码" | QLabel | Static text above password field |
| Password input | QLineEdit + QAction (eye icon) | EchoMode=Password, toggleable via eye icon |
| Error message | QLabel | Red text, initially hidden; shown on decrypt failure |
| Confirm button | QPushButton | Enabled when password non-empty |
| Cancel button | QPushButton | Calls reject(), dialog closes |

**States:**

| State | Password Input | Error Message | Confirm Button |
|-------|---------------|---------------|----------------|
| Initial | Empty | Hidden | Disabled |
| Typing | Content | Hidden | Enabled |
| Decrypt failure | Content (retained, cleared) | Red: "密码错误，请重试" | Enabled (for retry) |
| After max failed attempts (no limit) | Content (retained) | Error stays visible | Enabled (unlimited retry) |

### Dialog: PasswordDialog (另存为/更换密码)

**Mode:** `CHANGE_PASSWORD`

| Element | Type | Behavior |
|---------|------|----------|
| "更换密码" checkbox | QCheckBox | Unchecked by default; when unchecked, password fields hidden |
| Label: "新密码" | QLabel | Visible only when checkbox checked |
| Password input 1 | QLineEdit + eye icon | Visible only when checkbox checked |
| Label: "确认密码" | QLabel | Visible only when checkbox checked |
| Password input 2 | QLineEdit + eye icon | Visible only when checkbox checked |
| Password mismatch warning | QLabel | Same behavior as SET_PASSWORD mode, visible only when checkbox checked |
| Strength bar | QProgressBar | Same behavior, visible only when checkbox checked |
| "生成密码" button | QPushButton | Same behavior, visible only when checkbox checked |
| Confirm button | QPushButton | Always enabled (no password change required); if checkbox checked, validate same as SET_PASSWORD |
| Cancel button | QPushButton | Calls reject() |

**States:**

| State | Checkbox | Password Fields | Confirm Button |
|-------|----------|----------------|----------------|
| Initial (default) | Unchecked | Hidden | Enabled (save with same password) |
| Change requested | Checked | Visible, empty | Disabled until passwords valid |
| Change typed, valid | Checked | Visible, matching + >= 8 chars | Enabled |
| Change typed, invalid | Checked | Visible, mismatched or < 8 chars | Disabled |

### Sub-dialog: PasswordGenerator

| Element | Type | Behavior |
|---------|------|----------|
| Label: "密码长度：" | QLabel | Static |
| Length spinbox | QSpinBox | Range 8-128, default 16 |
| "大写字母" checkbox | QCheckBox | Checked by default |
| "小写字母" checkbox | QCheckBox | Checked by default |
| "数字" checkbox | QCheckBox | Checked by default |
| "符号" checkbox | QCheckBox | Checked by default; at least one charset must remain checked |
| Generated password preview | QLineEdge (read-only) | Shows generated password, updated on char change |
| "生成并使用" button | QPushButton | Closes sub-dialog, fills password into parent PasswordDialog fields |
| Cancel button | QPushButton | Closes sub-dialog without applying |

**Validation:** At least one character set must be selected. If user unchecks the last remaining set, prevent the action and show brief indicator.

**State table:**

| Selection State | Generate Button | Preview |
|----------------|----------------|---------|
| At least 1 charset checked | Enabled | Shows generated pw |
| All 4 unchecked | Disabled | Shows placeholder: "请至少选择一种字符类型" |

### Recent Files (WelcomeWidget)

| Element | Type | Behavior |
|---------|------|----------|
| Section header: "最近打开的文件" | QLabel | Static heading above list |
| Recent file list | QListWidget (or QLabel list) | Shows up to 5 file paths; click triggers open flow |
| No recent files state | QLabel | "暂无最近打开的文件" (gray text, shown when list empty) |

**States:**

| State | Visual |
|-------|--------|
| Loading | ListWidget populated from QSettings; stale entries (file not exist) removed silently |
| 0 items | "暂无最近打开的文件" label visible |
| 1-5 items | Clickable file entries; clicking triggers PasswordDialog open flow |
| Item clicked | QListWidget.itemClicked signal → MainWindow._on_open_recent(path) |

### Close Confirmation

| Element | Type | Behavior |
|---------|------|----------|
| Dialog title | QMessageBox icon=Question | "未保存的更改" |
| Message text | QMessageBox text | "笔记本有未保存的更改。是否在关闭前保存？" |
| Button 1 | QPushButton (Save) | "保存(S)" with accept role → trigger save, then close |
| Button 2 | QPushButton (Discard) | "不保存(D)" with destructive role → close without save |
| Button 3 | QPushButton (Cancel) | "取消(C)" with reject role → do nothing, stay open |

**State conditions:** Dialog only appears when `_is_dirty == True`. A new unsaved notebook with no modifications does not trigger this dialog.

## Accessibility

| Concern | Implementation |
|---------|---------------|
| Password visibility toggle | Eye icon QAction switches QLineEdit.EchoMode between Password and Normal |
| Keyboard navigation | All dialogs use Qt default tab order; QDialogButtonBox with keyboard shortcuts (&S, &D, &C) |
| Error messaging | Visual red text for all error states; no audio-only feedback |
| Password field clearance | On decrypt failure, password field contents are programmatically cleared; caller uses `bytearray` for secure zeroing |

---

## Security Considerations (UI Layer)

| Concern | Implementation |
|---------|---------------|
| Password in memory | PasswordDialog returns password via `password()` getter; caller reads once, calls explicit `clear_password()` afterward |
| Secure buffer zeroing | Password stored as `bytearray` internally in dialog; zeroed on dialog close |
| Password not in logs | Never printed or logged at any stage |
| File path from QSettings | Recent file paths read from QSettings, validated for existence before display |
